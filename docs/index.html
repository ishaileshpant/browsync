Cross-browser data consolidation

A Rust CLI toolchain that unifies bookmarks, browsing history, and login metadata across Chrome, Firefox, Safari, Edge, Brave, and Arc into a single searchable database.

Capabilities

Multi-browser import

Reads native data from Chrome (JSON), Firefox (SQLite), Safari (plist), and all Chromium derivatives directly. No manual export needed.

Full-text search

FTS5-indexed SQLite database. Prefix matching and multi-word queries across URLs, titles, folders, and tags.

Format-agnostic export

Netscape HTML (universally importable), Chrome-compatible JSON, and CSV with proper escaping. Filter by source browser.

Terminal UI

Ratatui-based interface with bookmark tree, history timeline, live search, and direct URL opening. Vim-style navigation.

Background daemon

File-system watcher on browser profiles. Debounced incremental sync with periodic full sweeps. Unix socket IPC.

Auth migration

Extracts login metadata (domain + username only) from credential stores. Generates migration reports between browsers.

Import pipeline

1
Read

Parser reads browser profile: Chrome Bookmarks (JSON), History (SQLite), Firefox places.sqlite, Safari Bookmarks.plist. Copies locked files to temp before reading.

2
Normalize

Converts browser-specific formats (WebKit timestamps, plist structures, Firefox IDs) into unified Bookmark and HistoryEntry models with UUID primary keys.

3
Deduplicate

Groups by URL. Same URL from multiple browsers merges into one entry. Last-write-wins for metadata, union-merge for tags. Visit counts are summed.

4
Store

Upserts into SQLite with FTS5 triggers. Full-text indexes on url, title, folder_path, and tags. Sync operation logged with timestamp and item count.

Command reference

CommandDescription
browsync detectScan for installed browsers and report available data.
browsync import [--browser <name>]Import bookmarks and history. Flags: --bookmarks-only, --history-only.
browsync search <query>Full-text search. Flags: --bookmarks, --history, -n <limit>.
browsync export --format <fmt>Export to html / json / csv. Optional --output and --browser.
browsync open <url> [--browser]Open a URL in a specific browser.
browsync tuiLaunch interactive terminal interface.
browsync statusDatabase stats and recent sync log.
browsync auth listList saved login domains (no passwords).
browsync auth migrate --from --toSites that need re-login when switching browsers.
browsyncd start|stop|statusBackground sync daemon control.

Browser extension

Real-time bookmark and tab sync via WebExtension Native Messaging.

# Build the extension
$ cd extension && npm install && npm run build

# Chrome: chrome://extensions > Developer Mode > Load unpacked
# Firefox: about:debugging > Load Temporary Add-on

# Install native messaging host (macOS)
$ cp extension/native-messaging-host.json \
    ~/Library/Application\ Support/Google/Chrome/NativeMessagingHosts/com.browsync.daemon.json
