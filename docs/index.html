Cross-browser data consolidation

A Rust CLI toolchain that unifies bookmarks, browsing history, and login metadata across Chrome, Firefox, Safari, Edge, Brave, and Arc into a single searchable database.

v0.1.0 Rust · Edition 2024 46 tests passing macOS / Windows / Linux
$ curl -fsSL https://ishaileshpant.github.io/browsync/install.sh | bash
PS> irm https://ishaileshpant.github.io/browsync/install.ps1 | iex
$ cargo install --git https://github.com/ishaileshpant/browsync browsync-cli

Capabilities

  • Multi-browser import

    Reads native data directly from Chrome (JSON), Firefox (SQLite), Safari (plist), and all Chromium derivatives. No manual export step.

  • Full-text search

    FTS5-indexed SQLite database. Prefix matching, multi-word queries across URLs, titles, folder paths, and tags.

  • Format-agnostic export

    Netscape HTML (universally importable), Chrome-compatible JSON, and CSV. Filter by source browser.

  • Terminal UI

    Ratatui-based interface with bookmark tree, history timeline, live search, and direct URL opening. Vim-style navigation.

  • Background daemon

    File-system watcher on browser profile directories. Debounced incremental sync with periodic full sweeps. Unix socket IPC.

  • Auth migration

    Extracts login metadata (domain + username, never passwords) from browser credential stores. Generates migration reports between browsers.

Three Rust crates plus a Manifest V3 browser extension.

Command reference

All subcommands and their options.

CommandDescription
browsync detectScan for installed browsers and report available data (bookmarks, history, credentials).
browsync import [--browser <name>]Import data from detected browsers. Supports --bookmarks-only and --history-only flags.
browsync search <query>Full-text search across all imported data. --bookmarks / --history to filter. -n to limit results.
browsync export --format <fmt>Export bookmarks to html, json, or csv. Optional --output <path> and --browser filter.
browsync open <url> --browser <name>Open a URL in a specific browser via the system launcher.
browsync tuiLaunch the interactive terminal interface.
browsync statusDisplay database statistics and recent sync operations.
browsync auth listList saved login domains and usernames across all browsers (no passwords stored).
browsync auth migrate --from <a> --to <b>Report which sites need re-login when switching from browser A to B.
browsyncd start | stop | statusControl the background sync daemon.

Browser extension

Real-time bookmark and tab sync via WebExtension Native Messaging (Manifest V3).

# Build
$ cd extension && npm install && npm run build

# Chrome: chrome://extensions > Developer Mode > Load unpacked
# Firefox: about:debugging > Load Temporary Add-on

# Install native messaging host (macOS)
$ cp extension/native-messaging-host.json \
    ~/Library/Application\ Support/Google/Chrome/NativeMessagingHosts/com.browsync.daemon.json
